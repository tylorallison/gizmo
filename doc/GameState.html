GameState

A generic game state class that provides building blocks for game state transitions. For example, a title screen, a main menu screen, and the main game play scene can all be managed by separate states. Each state can manage UI elements, handle player inputs, and setup event handlers that will only be active when the state has active and in a 'started' state. The Game class includes a StateMgr that is used to keep an inventory of available states and which state is currently active. Only one state can be active at a time. The game state has an internal state to track the progression of the state. Possible internal state values are as follows:

  • none: starting state, transitions to initialized during state start
  • initialized: state has been initialized, transitions to started during state start
  • started: state has been started, transitions to initialized if game state is stopped
